# Networking (How the internet talks)

## OSI 7 Layer Model
The OSI (Open Systems Interconnection) Model is a framework that explains how data travels across a network

It divides network communication into 7 layers, each with specific function

```
7. Application
6. Presentation
5. Session
4. Transport
3. Network
2. Data link
1. Physical
```
Data moves down the layers when sending and up the layers when receiving

## 7 - Aplication
The Aplication layer is where users interact with network services
Examples of protocol :
- HTTP
- HTTPS
- FTP
- DNS
- SMTP

Example command :
```
curl google.com
```
Explanation :<br>
This command sends an HTTP request to web server.

## Layer 6 - Presentation
The Presentation layer is responsible for :
- Data formating
- Encryption
- Compression
Examples :
- SSL/TLS encryption
- Data encoding<br>
Examples :<br>
`HTTPS uses TLS encryption`

# Layer 5 - Session
The Session layer manages connections between systems
Reponsibilities :
- Session establishment
- Session maintenance
- Session termination<br>
Example :<br>
`A user login session on a website`

# Layer 4 - Tranport
The Transport layer ensures reliable data delivery<br>
Main protocols :
- TCP
- UDP<br>

## Layer 3 - Network
The Network layer is responsible for routing data between networks<br>
Main Protocol
- IP (Internal Protocol)<br>
Example command :
`ip route`<br>
Example output :<br>
`default via 192.168.1.1 dev eth0`<br>
Explanation :<br>
Packets are sent through a gateway router

## Layer 2 - Data Link
The Data Link layer handles communication within the same network<br>
Examples :
- Eternet
- MAC addresses
- Switching<br>
Example command :
`ip link`<br>
This shows network interfaces

## Layer 1 - Physical
The Physical layer represents the hardware that transmits data<br>
Example :
- Eternet cable
- Fiber optic cable
- Wifi signals<br>
This layer send raw bits (0 and 1) over the medium

## Example : Opening a Website
When you open :<br>
`https://google.com`
The layers work like this :
```
Application - HTTP request
Presentation - TLS encryption
Session - connection session
Transport - TCP connection
Network - IP routing
Data Link - Ethernet frame
Physical - electrical signal
```
